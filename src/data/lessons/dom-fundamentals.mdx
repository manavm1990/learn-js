# DOM Fundamentals

## Introduction

The browser takes your HTML and converts it into the **DOM**—a tree of JavaScript objects. Each HTML tag becomes an object with properties you can read and methods you can call.

Understanding the DOM means understanding that **your HTML is not the same as what JavaScript sees**. The browser parses HTML, fixes errors, and builds a live object structure that you can inspect and manipulate.

This lesson focuses on what the DOM *is* and how to work with DOM objects. The next lesson will focus on *finding* those objects efficiently.

## Learning Objectives

By the end of this lesson, you'll be able to:

- Understand the DOM as a tree of JavaScript objects
- Know the difference between HTML source code and the live DOM
- Access DOM objects via `document` and basic selectors
- Read and change properties of DOM elements (`textContent`, `innerHTML`, `classList`)
- Add event listeners to respond to user interaction
- Create and insert new elements into the page
- Inspect the DOM using Developer Tools

## What Is the DOM?

**DOM** = Document Object Model. It's the browser's internal representation of your HTML as JavaScript objects.

**HTML is text.** The browser reads it, parses it, builds a tree of objects.

### Simple Example

**Your HTML:**

```html
<div>
  <h1>Hello</h1>
  <p>Welcome to the DOM.</p>
</div>
```

**The DOM (as JavaScript objects):**

```javascript
document.body = {
  tagName: "BODY",
  textContent: "Hello Welcome to the DOM.",
  children: [
    {
      tagName: "DIV",
      children: [
        { tagName: "H1", textContent: "Hello" },
        { tagName: "P", textContent: "Welcome to the DOM." },
      ],
    },
  ],
}
```

Each element is an object. Each object has:
- **Properties** (data): `textContent`, `className`, `id`, `children`, etc.
- **Methods** (actions): `addEventListener()`, `appendChild()`, `querySelector()`, etc.

### Key Difference: HTML vs DOM

**HTML source code:**
```html
<p>Hello</p>
```

**DOM object for that element:**
```javascript
{
  tagName: "P",
  textContent: "Hello",
  className: "",
  id: "",
  // ... 50+ other properties and methods
}
```

The browser **fixes your HTML** before building the DOM. Missing tags are auto-closed. Errors are corrected. That's why the DOM might look different from your source HTML.

## Accessing the DOM

The `document` object is your entry point to the DOM:

```javascript
// Access the entire document
console.log(document);

// Access the body element
console.log(document.body);

// Access the root html element
console.log(document.documentElement);

// Find elements (we'll cover this deeply in the next lesson)
const title = document.getElementById("page-title");
const buttons = document.querySelectorAll("button");
```

For now, just know: **`document` is a global object** that represents your entire page. Everything starts there.

## DOM Objects Have Properties and Methods

Every DOM element is a JavaScript object. Here's what that means:

### Properties (Data About the Element)

```javascript
const heading = document.querySelector("h1");

// Properties you can READ
console.log(heading.textContent);  // The text inside
console.log(heading.innerHTML);    // HTML inside the element
console.log(heading.id);           // The id attribute
console.log(heading.className);    // The class attribute
console.log(heading.tagName);      // "H1"
console.log(heading.children);     // Child elements (array-like)
console.log(heading.parentElement);// Parent element

// Properties you can CHANGE
heading.textContent = "New title";
heading.className = "active";
```

### Methods (Actions You Can Perform)

```javascript
const button = document.querySelector("button");

// Listen for an event
button.addEventListener("click", () => {
  console.log("Button was clicked!");
});

// Modify the element
button.textContent = "Clicked!";

// Add a child element
const newDiv = document.createElement("div");
button.appendChild(newDiv);

// Search inside this element
const firstParagraph = button.querySelector("p");
```

**Key insight:** Properties hold data. Methods perform actions. Both are critical to working with the DOM.

## Changing Element Content

### `textContent` - Change Text Only

```javascript
const heading = document.querySelector("h1");

// textContent is a PROPERTY on the DOM element object
heading.textContent = "New heading text";

// Safe from XSS attacks - HTML tags become literal text
heading.textContent = '<script>alert("hack")</script>'; // Shows as text, not code
```

### `innerHTML` - Change HTML Content

```javascript
const container = document.querySelector(".content");
container.innerHTML = "<p>New <strong>paragraph</strong> with HTML</p>";

// ⚠️ Be careful - can be dangerous with user input
// Only use with content you control
```

### Practical Example

```html
<div id="user-info">
  <h2>Loading...</h2>
  <p>Please wait</p>
</div>
```

```javascript
// Update when data loads
const userInfo = document.getElementById("user-info");
userInfo.innerHTML = `
  <h2>Welcome, ${userName}!</h2>
  <p>Last login: ${lastLogin}</p>
`;
```

## Adding Event Listeners

DOM objects can listen for user interaction. Every time something happens (click, type, hover), the browser sends an **event**. You can catch it with `addEventListener()`:

```javascript
const button = document.querySelector("button");

button.addEventListener("click", () => {
  console.log("User clicked the button!");
});

const input = document.querySelector("input");

input.addEventListener("input", () => {
  console.log("User typed:", input.value);
});
```

**Common events:**
- `click` - user clicked
- `input` - user typed in a form field
- `submit` - user submitted a form
- `mouseenter` / `mouseleave` - user hovered
- `change` - value changed

When an event happens, the listener function runs. More on this in later lessons.

## Working with CSS Classes (Tailwind Focus)

Instead of changing styles directly, you add/remove CSS classes. The `classList` property makes this easy:

```javascript
const card = document.querySelector(".card");

// classList is a PROPERTY on the DOM element
// It has METHODS: add, remove, toggle, contains

// Add Tailwind classes
card.classList.add("ring-2", "ring-blue-500");

// Remove classes
card.classList.remove("opacity-50");

// Toggle (add if missing, remove if present)
card.classList.toggle("hidden");

// Check if class exists
if (card.classList.contains("active")) {
  console.log("Card is active");
}
```

### Why This Matters

**Old way (inline styles):**
```javascript
card.style.backgroundColor = "blue";
card.style.padding = "1rem";
card.style.borderRadius = "0.5rem";
```

**Tailwind way (add/remove classes):**
```javascript
card.classList.add("bg-blue-500", "p-4", "rounded");
```

The Tailwind approach keeps styling logic in HTML/CSS, not scattered in JavaScript.

## Creating and Adding Elements

Sometimes you need to create new DOM objects and insert them into the page:

```javascript
// Create a new element object
const newDiv = document.createElement("div");

// Set its properties
newDiv.textContent = "Hello from JavaScript!";
newDiv.className = "bg-blue-100 p-4 rounded";

// Add it to the page
const container = document.querySelector("#container");
container.appendChild(newDiv);
```

**Key methods:**
- `document.createElement(tagName)` - Create a new element object
- `element.appendChild(child)` - Add a child element
- `element.removeChild(child)` - Remove a child element

### Practical Example: Building a Card

```javascript
function createCard(title, description) {
  // Create the card element
  const card = document.createElement("div");
  card.classList.add("bg-white", "rounded", "shadow", "p-6", "mb-4");

  // Create and add a heading
  const heading = document.createElement("h3");
  heading.textContent = title;
  heading.className = "text-xl font-bold mb-2";
  card.appendChild(heading);

  // Create and add a paragraph
  const para = document.createElement("p");
  para.textContent = description;
  para.className = "text-gray-600";
  card.appendChild(para);

  return card;
}

// Use it
const cardList = document.querySelector("#cards");
const myCard = createCard("DOM Basics", "Learn how JavaScript interacts with HTML");
cardList.appendChild(myCard);
```

This pattern is the foundation of dynamic page generation.

## Inspecting the Live DOM with Developer Tools

### Viewing the Live DOM

1. **Open any webpage**
2. **Press F12** to open Developer Tools
3. **Click the Elements or Inspector tab**

What you see is the **live DOM**—not your original HTML source code.

### Key Difference: Source HTML vs Live DOM

**Your original HTML file:**
```html
<div>
  <p>Hello world</p>
</div>
```

**The browser's DOM (what you see in DevTools):**
```html
<html>
  <body>
    <div>
      <p>Hello world</p>
    </div>
  </body>
</html>
```

The browser **auto-corrected** your HTML by adding missing tags.

### Testing DOM Changes in Real Time

Open any webpage, press F12, and try this in the Console:

```javascript
// Get an element
const heading = document.querySelector("h1");

// Change its text
heading.textContent = "I changed this!";

// Add a class
heading.classList.add("text-red-500");

// Look at the Elements tab — you'll see your changes!
```

Refresh the page — changes disappear (they were only in the DOM, not saved to disk).

---

## Summary: The DOM is Live

When you modify a DOM object, the browser immediately updates what the user sees. No refresh needed. This is the power of the DOM.

## Practical DOM Exploration

Let's build a simple example that demonstrates DOM objects in action:

### HTML

```html
<main>
  <h1 id="title">DOM Explorer</h1>
  <button id="inspect-btn">Inspect This Page</button>
  <div id="output"></div>
</main>
```

### JavaScript

```javascript
const inspectBtn = document.getElementById("inspect-btn");
const output = document.getElementById("output");

inspectBtn.addEventListener("click", () => {
  // Get DOM objects
  const title = document.getElementById("title");
  const button = document.getElementById("inspect-btn");

  // Read their properties
  const results = [
    `Title text: "${title.textContent}"`,
    `Title tag name: ${title.tagName}`,
    `Button has ${button.children.length} child elements`,
    `Output div is empty: ${output.children.length === 0}`,
    `Page has ${document.querySelectorAll("*").length} total elements`,
  ];

  // Build output element
  const list = document.createElement("ul");
  results.forEach((result) => {
    const item = document.createElement("li");
    item.textContent = result;
    item.className = "p-2 bg-gray-100 mb-1 rounded";
    list.appendChild(item);
  });

  output.appendChild(list);
});
```

### What's Happening

1. We find DOM objects using `getElementById()` and `querySelectorAll()`
2. We read properties from those objects (`textContent`, `tagName`, `children.length`)
3. We create new DOM objects and add them to the page (`createElement()`, `appendChild()`)
4. The page updates instantly—no refresh needed

This is the DOM in action.
