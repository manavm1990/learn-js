# Learn Markdown

## What is Markdown?

Markdown is a lightweight markup language that you can use to add formatting elements to plaintext text documents. The key benefits of using Markdown include:

- **Simplicity**: Easy to learn and use, with a straightforward syntax.
- **Readability**: Plain text files are easy to read and edit without specialized software.
- **Versatility**: Can be converted to HTML and other formats, making it ideal for web content.

You will quickly notice that this is like a light üö® version of HTML. The difference is that this cannot be rendered directly with a browser. Instead we need a markdown renderer. In addition, the styles üíÑ are generally set by the aforementioned rendered. Summarily, this is a great tool for creating structured documents without the complexity of traditional word processors or HTML/CSS üíÑ.

## What Markdown Isn't

Before we go further, it helps to know what Markdown is NOT:

- It is not HTML. Browsers do not understand `.md` files the way they understand `.html` files.
- It is not a programming language. Markdown has no logic or variables‚Äîit's just text with simple formatting rules.
- It is not a WYSIWYG (What You See Is What You Get) document like Google Docs. You write plain text and a renderer turns it into styled output.
- It does not load in a browser by itself. If you open a `.md` file in a browser, you'll typically just see plain text unless a site has a Markdown renderer (for example, GitHub does).

Think of Markdown as ‚Äúnicely formatted notes‚Äù that become pretty only after a helper tool converts them.

## File extensions: `.txt` vs `.md` vs `.html`

File extensions tell computers and browsers how to treat a file:

- `.txt` ‚Äî Plain text. No formatting rules. A browser or text editor shows the text exactly as written.
- `.md` ‚Äî Markdown text. Still plain text, but with special symbols (like `#`, `*`, `[ ]`) that a Markdown renderer can turn into headings, lists, and links.
- `.html` ‚Äî HyperText Markup Language. Browsers natively understand and render HTML, applying structure and styles.

Practical differences:

- Double-clicking `notes.txt` opens a text viewer. No styling.
- Double-clicking `readme.md` opens a text viewer (or a specialized app). In a browser without a renderer, it looks like plain text. On GitHub, it looks styled because GitHub renders it for you.
- Double-clicking `index.html` opens in your browser and renders with headings, paragraphs, images, etc., because browsers speak HTML.

## Basic Syntax

Here are some common Markdown syntax elements:

### Headers

Use `#` for headers. The number of `#` symbols indicates the header level.

```markdown
# Header 1

## Header 2

### Header 3
```

### Emphasis

- _Italic_: Use single asterisks or underscores.
- **Bold**: Use double asterisks or underscores.

```markdown
_Italic_ or _Italic_
**Bold** or **Bold**
```

### Lists

- **Unordered lists**: Use asterisks, plus signs, or hyphens. **Hot ‚ô®Ô∏è tip:** You can indent subitems with two spaces or a tab!
- **Ordered lists**: Use numbers followed by periods. **Hot ‚ô®Ô∏è tip:** You can number everything with `1.` and Markdown will auto-number it for you!

```markdown
- Item 1
- Item 2
  - Subitem 1
  - Subitem 2

1. First item
2. Second item
```

### Links

Create links using the following syntax:

```markdown
[Link text](URL)
```

Example: `[Google](https://google.com)` becomes [Google](https://google.com)

### Images

Images are similar to links but with an exclamation mark in front:

```markdown
![Alt text](Image URL)
```

Example: `![Cat photo](https://example.com/cat.jpg)`

### Code

You can show code inline or in blocks:

- **Inline code**: Wrap text in single backticks: `` `code here` ``
- **Code blocks**: Use three backticks before and after:

````markdown
```
const greeting = "Hello, world!";
```
````

**Pro tip**: Add a language name after the opening backticks for syntax highlighting:

````markdown
```javascript
const greeting = "Hello, world!";
```
````

### Blockquotes

Use `>` to create a blockquote:

```markdown
> This is a quote.
```

Renders as:

> This is a quote.

## Common Mistakes

Here are some common mistakes to avoid when using Markdown:

1. **Forgetting to close emphasis tags**: Ensure you close your italic or bold tags correctly. For example, use `_italic_` instead of `_italic`.
2. **Incorrectly formatting lists**: Be consistent with your list formatting. Mixing ordered and unordered lists can lead to confusion.
3. **Not using the correct file extension**: Always save your Markdown files with the `.md` extension to ensure proper rendering.
4. **Missing blank lines**: Many Markdown renderers need blank lines before and after headings, lists, and code blocks. When in doubt, add a blank line!

## Preview Your Markdown

Since browsers don't render `.md` files directly, you need a tool to preview your work:

- **GitHub Gist**: Automatically previews `.md` files (as you saw in the experiment above)
- **VS Code**: Built-in Markdown preview (right-click a `.md` file ‚Üí "Open Preview")
- **Dillinger**: Online editor with live preview at [dillinger.io](https://dillinger.io/)
- **Notion**: Supports Markdown shortcuts
- **Google Docs**: [Can enable Markdown](https://support.google.com/docs/answer/12014036?hl=en)

**Why so many tools?** Different developers prefer different workflows. Some like online tools (Gist, Dillinger), while others prefer working locally (VS Code). Try a few and see what fits your style!
